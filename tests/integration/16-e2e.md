# `ca-usb-run` orchestrator end-to-end (test gate)

The full signed-job lifecycle, wired by `box/bin/ca-usb-run`. This doc
records what is proven AUTOMATICALLY today and the physical
stick/K1/real-root-dispatch parts that are **deferred to the operator** (they
need a real vfat block device to `mount`, a real finger on K1, and root
`setpriv` — none of which a unit test or a non-root box session can stand in
for).

---

## A. Automated coverage (runs on the box today — the real integration)

`tests/unit/test_ca_usb_run.py::TestLifecycle` drives the ORCHESTRATOR through
its whole state machine with the REAL `verify` / `extract` (openat2) /
`manifest` / `freshness` / `collect` / `commitlog` modules against
ssh-keygen-signed crafted `job.tar`s and injected tmpdir anchors/state. Only
the genuinely un-unit-testable edges are mocked: the block-device
mount/umount (`mountctl.*`), the K1 evdev button, the LED sysfs writes,
`sync`, and — for the run-script happy path — `dispatch.run` (a real
run-script needs root `setpriv`; its own real behaviour is
`tests/unit/test_dispatch.py`'s job). The cosign_failed negative uses the
**real** `dispatch.run`, which raises before any exec, so "nothing runs as
root" is genuine.

Covered end to end (each `... ok` in the suite):

- **Happy path** — signed `run-script` echo job → verify→extract→freshness→K1
  press→dispatch→collect→commit→deliver → `outbox/<job_id>/{status.json,
  result.log}` + top-level `outbox/LATEST.json`; `status:ok`, `replayed:false`,
  `result.log` sha256 matches `status.json.outputs[]`; committed on-box
  (`results/<job_id>/DONE`), `seq` bumped; LED
  `VERIFYING→READY→RUNNING→SAFE_REMOVE`.
- **Replay** — same `job_id`, higher `seq` (`caj --retry`) → the
  IDENTICAL cached `result.log` bytes redelivered with `status.replayed:true`
  and the CACHED `seq`; the handler does NOT re-run (asserted via a sentinel
  `dispatch.run`); replay with no K1 leaves the cache and delivers nothing.
- **Negatives, each fail-closed with the right error enum, nothing runs,
  nothing committed pre-freshness** — unsigned (`verify_failed`), wrong-key sig
  (`verify_failed`), `seq<=last` (`stale_seq`), `box≠nebula-ca` (`wrong_box`),
  `jobs>1` (`jobs_gt_1`), malformed manifest (`bad_manifest`), privileged
  run-script without a break-glass co-sign (`cosign_failed`, real dispatch,
  nothing runs as root).
- **Break-glass co-sign positive** — distinct operator+breakglass keys, a valid
  `job.tar.bg.sig` → the real 4-arg `verify.verify_cosign` passes → the
  orchestrator passes `cosigned=True` (a real `bool`) into dispatch.
- **no_confirmation** — K1 timeout → committed (seq consumed) +
  delivered `status:no_confirmation`, `presence_confirmed:false`,
  `exit_code:null`, handler never called.
- **handler rc≠0** — `status:error` / `error:handler_failed`, committed +
  delivered.
- **Recovery branch** — blank stick (no `inbox/job.tar`) → K1-gated
  `recovery.write`; one press ⇒ `confirm2=False` (public-only), a distinct
  second press ⇒ `confirm2=True` (registry opt-in), declined (no press) ⇒
  nothing written; a PRESENT-but-bad job fails closed, NEVER recovery; LED
  `RECOVERY_OFFER→RECOVERY_CONFIRM2→RECOVERY_WRITE`.
- **Ordering guarantee (durability crux)** — a `commit` that raises leaves the
  outbox empty and never remounts rw (deliver never precedes commit); a
  deliver failure AFTER commit leaves the on-box committed results intact for a
  future retry.

### A.1 caj → orchestrator → outbox wire round-trip (recorded)

A REAL `mac/caj build` job (not a hand-built tar) driven through the
orchestrator on the box, mount/K1/LED/sync mocked, dispatch simulated
(run-script needs root):

```
$ python3 e2e_caj_orchestrator.py
[e2e] caj: built job 209e17a3-40ff-424c-aab7-00488464364e (seq=1) -> .../stick/inbox
[e2e] orchestrator rc=0  LED=['VERIFYING', 'READY', 'RUNNING', 'SAFE_REMOVE']
[e2e] delivered LATEST.json: {'box':'nebula-ca','bundle_id':'209e17a3-...',
      'job_id':'209e17a3-...','replayed':False,'seq':1,'status':'ok'}
[e2e] result.log sha256 matches status.json (15 bytes): OK
[e2e] PASS (caj-built job -> orchestrator -> verified outbox; cleaned up)
```

This proves the Mac tooling's `job.tar`/`job.tar.sig`/`manifest.json` wire
format is consumable by the box unchanged (verify + hardened extract + strict
manifest + freshness all pass on caj's real output). `caj-recv`'s own outbox
validation is covered by `tests/unit/test_caj_recv.py` and was NOT invoked here
because it writes into its own repo `ca-state/` (would mutate the source tree).

## B. Deferred to the operator (real stick + real K1 + root, on the box)

These require the physical ceremony and cannot be stood in for. Run them once
before air-gap as part of the test gate.

### B.1 Happy path with a real stick + real K1
- [ ] `caj build --spec <job.spec> --stick /Volumes/CA-XFER` for a `run-script`
      echo job (or a vetted `status` job); insert the stick into the box.
- [ ] udev fires `ca-usb-job@sda1` → `user_led` (GREEN) shows **VERIFYING**
      (~1.5 Hz) then **READY** (fast ~4 Hz "act now") — press **K1** (NOT the
      MASK button) once. (See the LED-key sticker; rates revised 2026-07-16.)
- [ ] `user_led` **RUNNING** (solid) then held **SAFE_REMOVE** (slow, calm
      ~0.5 Hz) only after the box has unmounted. Pull the stick → **IDLE** (off).
- [ ] On the Mac: `caj-recv --stick /Volumes/CA-XFER` verifies
      `outbox/LATEST.json` + `outbox/<job_id>/status.json`, recomputes every
      `outputs[].sha256`, and advances `ca-state/last-seq` from the box `seq`.

### B.2 Replay (`caj --retry`)
- [ ] Re-run B.1 but on the Mac `caj build --retry <job_id> ...` (same
      `job_id`, new `seq`); insert + K1. The box redelivers the **identical**
      cached bytes with `status.replayed:true` and the handler does NOT re-run.
- [ ] A same-stick reinsert (unchanged `seq`) is rejected as `stale_seq`.

### B.3 Negative matrix (real stick)
- [ ] Unsigned job (delete `job.tar.sig`) → **ERROR** LED, no outbox, nothing
      committed. Wrong-key sig → same.
- [ ] `seq<=last` → ERROR (`stale_seq`); `box` ≠ `nebula-ca` → ERROR
      (`wrong_box`); `jobs>1` / malformed `manifest.json` → ERROR.
- [ ] A privileged `run-script` (`args.privileged: true`) WITHOUT a valid
      `inbox/job.tar.bg.sig` → **ERROR after K1** (`cosign_failed`); confirm
      via `journalctl -u ca-usb-job@*` that nothing ran as root. Build this
      case WITHOUT `--breakglass` (`caj` emits `job.tar.bg.sig` via
      `caj build --breakglass <breakglass-key>` — omit the flag
      here on purpose). (Note: `args.privileged: true` reaches the box as a
      real JSON boolean only as of a caj fix — caj
      used to store every `args.<name>` value as a plain string, so this
      exact spec line would previously have silently run UNPRIVILEGED
      instead of hitting `cosign_failed`.)

### B.4 Recovery kit (blank stick)
- [ ] Insert a blank vfat stick (no `inbox/job.tar`) → LED **RECOVERY_OFFER**
      (distinct from READY, ~0.67 Hz). One K1 press → **RECOVERY_WRITE** →
      `<stick>/CA-RECOVERY/` written (public-only). Verify **no** `ca.key` and
      **no** `registry.json` on the stick.
- [ ] Repeat, giving a DISTINCT second K1 press within the confirm2 window
      (LED **RECOVERY_CONFIRM2**, ~3.3 Hz) → `registry.json` IS included.
- [ ] Blank stick + no K1 within the window → nothing written; safe to remove.

### B.5 Real run-script dispatch as `nebula-job` (root path)
- [ ] With the harness running as root (systemd), a non-privileged `run-script`
      actually drops to `nebula-job` via `setpriv` (verify `id` in the script's
      output), cannot read `ca.key`, and its outputs are collected
      symlink-safely. (Automated coverage mocks `dispatch.run`; the real
      privilege drop is `tests/integration/14-dispatch.md` /
      `dispatch_root.py`.)
